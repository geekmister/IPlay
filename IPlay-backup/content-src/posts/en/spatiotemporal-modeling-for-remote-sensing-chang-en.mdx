# Spatiotemporal Modeling for Remote Sensing Change Detection

> Perspective: False alarm cost should be modeled explicitly instead of optimizing recall alone.

## Research Question

This article focuses on remote sensing change detection: improving interpretability, stability, and deployability while preserving strong performance.

## Method Perspective

1. Define task constraints before increasing model complexity.
2. Use both perceptual and objective metrics for evaluation.
3. Replay failure cases during training to reduce tail-risk.

## Evaluation Suggestions

- Report not only peak scores, but also variance and worst-case behavior.
- Add cross-domain validation to avoid single-dataset overfitting.
- Include latency and memory costs for engineering decisions.

## Representative Papers and Links

- [SwinIR (2021)](https://arxiv.org/abs/2108.10257)
- [BiSeNet (2018)](https://arxiv.org/abs/1808.00897)
- [NTS-Net (2018)](https://arxiv.org/abs/1809.00287)

## Production Insight

False alarm cost should be modeled explicitly instead of optimizing recall alone. In practical delivery, I strongly recommend using a minimum loop of failure replay, metric dashboarding, and rollback plans.
